# Go-Live Readiness – Client Onboarding

Before moving to production, CMS vendors and courts must complete the following checks.

---

## ✔ Technical Readiness

- All tests successfully certified  
- All case types enabled and validated  
- All documents retrieve properly  
- All security rules match court policies  
- S3 or SOAP endpoints configured for production  

---

## ✔ Operational Readiness

- Court staff trained  
- Support contacts confirmed  
- Monitoring dashboards configured  
- Logging retention configured  
- SLA expectations defined  

---

## ✔ Deployment Checklist

### Batch Mode
- Production S3 keys provided  
- Production folder structure validated  
- File schedule confirmed  

### ECF Mode
- Production certificate installed  
- Production endpoints configured  
- Court codes validated  

---

## ✔ Tyler BIS Approvals

Final items required:

- TPM signoff  
- BIS support validation  
- Court approval  
- Vendor approval  

Once everything is approved → **production go-live scheduling begins**.
