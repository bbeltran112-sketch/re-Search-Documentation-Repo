## Folder Purpose Quick Reference

| Folder | Purpose | Who Uses It |
|--------|---------|-------------|
| `getting-started/` | Product introduction | Everyone |
| `core-product/api/` | API reference | Everyone |
| `core-product/integration/` | How to integrate | Everyone |
| `core-product/onboarding/` | Standard onboarding | Everyone |
| `core-product/support/` | General troubleshooting | Everyone |
| `markets/texas/` | Texas-specific requirements | Texas only |
| `markets/california/` | CA-specific requirements | California only |
| `markets/[other]/` | Other market requirements | That market only |

## Finding What You Need

### "I need to understand the API"
→ `core-product/api/`

### "I'm implementing for Texas"
→ Start with `core-product/`, then add `markets/texas/`

### "I'm implementing for [other state]"
→ Check if `markets/[state]/` exists
   - Yes: Use `core-product/` + `markets/[state]/`
   - No: Just use `core-product/`

### "I have a support issue"
→ Check `core-product/support/` first
→ If market-specific, check `markets/[your-market]/support/`

### "I need to know about JCIT"
→ `markets/texas/jcit-compliance/`
   (This is the ONLY place JCIT docs exist)

## Why This Structure Works

### Clear Mental Model
```
Universal stuff → core-product/
Specific stuff → markets/[market-name]/
```

### No Ambiguity
- If it's in `core-product/`, it applies to everyone
- If it's in `markets/texas/`, it's Texas-only
- No questions, no confusion

### Easy to Maintain
- Core product changes? Edit `core-product/`
- Texas regulations change? Edit `markets/texas/`
- New market? Add `markets/new-market/`
- No cross-contamination

### Scales Naturally
```
Current: 1 market with special requirements (Texas)
Future: Add markets/california/, markets/florida/, etc.
Core product stays clean regardless of markets added
```

## Navigation Tips

1. **Always start with the README**
   - Every folder has a README that tells you what's inside
   - READMEs guide you to next steps

2. **Follow the tier progression**
   - Don't skip to Tier 3 without doing Tier 1 & 2
   - Each tier builds on the previous

3. **Use the search function**
   - But now you know where things are
   - No more "is this JCIT for everyone or just Texas?"

4. **Bookmark key pages**
   - `core-product/api/` for daily reference
   - `markets/[your-market]/` if applicable
