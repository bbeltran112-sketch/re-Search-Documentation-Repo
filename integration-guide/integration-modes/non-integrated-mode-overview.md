# Non-Integrated Mode Overview

Non-Integrated Mode applies when a court has **no CMS integration** with re:Search.  
Case data is generated only from e-filing activity, and clerks must manually maintain parity.

## Appropriate For

- Pilot or low-volume courts  
- CMS systems that cannot integrate  
- Temporary fallback scenarios  

---

## Limitations

- No automated updates  
- Data may become outdated  
- No event-driven messaging
